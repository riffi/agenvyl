# Agenvyl v0.3.0

This release makes agent setup and day-to-day room work clearer, adds safer
OpenCode permission handling, and improves runtime recovery around workspaces
and local harness processes.

## What's new

- A streamlined response timeline with compact agent cards, collapsible run
  activity, direct links to changed file versions, clearer tool status, and
  more focused run actions.
- Better mobile room navigation, wider agent responses, refined responder
  chips, and a more compact agent catalog.
- Resilient harness discovery. Agenvyl caches successful model and control
  catalogs, keeps stale entries usable when a refresh fails, and provides an
  explicit refresh action.
- OpenCode **Standard** and **Auto-approve** permission profiles, structured
  approvals and multi-question clarifications, and more reliable completion,
  cancellation, and session cleanup.
- An explicit OpenCode external-directory allowlist. Access is denied by
  default; a validated directory can be configured on the instance or added
  through an approval.
- Clearer first-run controls for managed OpenCode, Codex danger-full-access,
  and experimental Claude subscription OAuth.
- More durable run-workspace cleanup and supervisor lifecycle handling,
  including quieter Windows uninstall and clearer installer progress.
- Opt-in operator modes for no-op workspace publication, warm run slots, and
  filesystem stat caching. They remain disabled by default.

## Upgrade notes

The standalone agent role field has been removed from the UI, API, and
database. Existing role text is discarded during migration rather than being
converted into agent instructions. Preserve it separately before upgrading if
it is still needed.

Database migrations run automatically at Core startup. Back up PostgreSQL and
the workspace root as one recovery point before upgrading a persistent
installation.

OpenCode external-directory access now has an explicit instance allowlist.
Existing instances start with an empty list, so external access remains denied
until a trusted root is deliberately added.

## Platforms and harnesses

Self-contained archives are provided for Linux x64/arm64, macOS x64/arm64, and
Windows x64. They bundle Node.js 22 and PostgreSQL 17.10.

The Connector supports Codex CLI, Claude Code, OpenCode, Antigravity, and
Hermes. Harness capabilities differ; see the
[capability matrix](../harnesses/capabilities.md) before choosing an agent.

## Trust boundary

This release is unsigned and intended for a trusted single-operator
workstation. It has no service/autostart, automatic update, tray application,
telemetry, or multi-user authorization layer. A room workspace and every
explicit OpenCode external-directory root are trusted agent-accessible
locations, not security sandboxes. Rendered HTML and SVG are isolated, but may
execute document content within the preview boundary. Review the
[trust guide](../user-guide/trust-and-security.md) before installation.

## Install

Linux or macOS:

```bash
curl -fsSL https://github.com/riffi/agenvyl/releases/download/v0.3.0/install.sh |
  sh -s -- --version 0.3.0
```

Windows PowerShell:

```powershell
$env:AGENVYL_VERSION = '0.3.0'
irm https://github.com/riffi/agenvyl/releases/download/v0.3.0/install.ps1 | iex
```
